🔍 Core Concept

The WHERE Clause

🧠 Logical Operators

AND, OR, NOT

Combine multiple conditions to build powerful, precise filters.

AND
Both must be true
Like requiring age > 18 AND city = 'Delhi'. Both conditions must be satisfied for a row to be returned.
WHERE age > 18 AND city = 'Delhi'
OR
Either can be true
Like showing students from Delhi OR Mumbai. If any one condition is true, the row is included.
WHERE age < 18 OR marks > 90
NOT
Reverses the condition
Excludes rows matching the condition. NOT city = 'Mumbai' gives everyone except Mumbai students.
WHERE NOT city = 'Mumbai'
📝 All Three Examples
SQL — Logical Operators
Copy
-- AND: age above 18 AND from Delhi
SELECT * FROM Students
WHERE age > 18 AND city = 'Delhi';

-- OR: younger than 18 OR scored above 90
SELECT * FROM Students
WHERE age < 18 OR marks > 90;

-- NOT: exclude Mumbai students
SELECT * FROM Students
WHERE NOT city = 'Mumbai';
🚀 Advanced Filtering

BETWEEN, IN, LIKE

More powerful filtering tools for ranges, lists, and pattern matching.

📏 BETWEEN — Range Filter
Selects rows where a value falls within a range (inclusive on both ends). Cleaner than writing age >= 18 AND age <= 25.
SQL
Copy
-- Age between 18 and 25 (inclusive)
SELECT * FROM Students
WHERE age BETWEEN 18 AND 25;

-- Same as: WHERE age >= 18 AND age <= 25
0
18
25
99
← range highlighted
📋 IN — List Filter
Checks if a value matches any item in a given list. Much cleaner than chaining multiple OR conditions.
With IN
SELECT * FROM Students
WHERE city IN ('Delhi', 'Mumbai');
Equivalent (messy)
WHERE city = 'Delhi'
OR city = 'Mumbai';
🔤 LIKE — Pattern Matching
Search for rows where a string matches a pattern. Uses two wildcards: % (any characters) and _ (exactly one character).
%
Percent — any number of characters
R% → Rohit, Raj, Riya, Ramesh... (starts with R)
_
Underscore — exactly one character
_a% → Raj, Kat, Sam... (second letter is 'a')
SQL — LIKE Examples
Copy
-- Names starting with 'R'
SELECT * FROM Students
WHERE name LIKE 'R%';

-- Names where second character is 'a'
SELECT * FROM Students
WHERE name LIKE '_a%';

-- Names ending with 'a'
SELECT * FROM Students
WHERE name LIKE '%a';

✂️ LIMIT

Control Output Size

LIMIT restricts the number of rows returned. Essential for pagination, top-N results, and preventing massive data dumps.

🎯 Interview Questions

Ace Your SQL Interview

These are frequently asked in technical interviews. Know them cold.

Q1
What is the difference between WHERE and HAVING?
▼
WHERE filters rows before grouping. It works on individual rows and cannot use aggregate functions like SUM(), COUNT().

HAVING filters groups after GROUP BY. It's used with aggregate functions.

-- WHERE (individual rows)
SELECT * FROM Sales WHERE amount > 1000;

-- HAVING (groups)
SELECT city, SUM(amount) FROM Sales
GROUP BY city HAVING SUM(amount) > 5000;
Q2
Difference between IN and BETWEEN?
▼
IN checks if a value matches any item in a discrete list: city IN ('Delhi', 'Mumbai', 'Pune'). Best for specific values you want to include.

BETWEEN checks if a value falls within a continuous range (inclusive): age BETWEEN 18 AND 25. Best for numeric or date ranges.

📌 Key: IN uses a list. BETWEEN uses a range. They're not interchangeable.
Q3
When do you use LIKE vs = ?
▼
Use = when you know the exact value: name = 'Rohit' — returns only exact match.

Use LIKE when you need pattern matching: name LIKE 'R%' — returns Rohit, Raj, Ram, Riya, etc.

⚠️ Performance note: = is faster than LIKE because LIKE can require scanning every character. Use = whenever you can.
Q4
Can WHERE be used without SELECT?
▼
Yes! WHERE works with UPDATE, DELETE, and other DML statements too.

-- Update only specific rows
UPDATE Students SET city = 'Delhi' WHERE id = 5;

-- Delete only specific rows
DELETE FROM Students WHERE age < 18;
⚠️ Always use WHERE with DELETE/UPDATE — without it, you'll modify ALL rows!
⚠️ Common Mistakes

Don't Get Caught by These

❌ Using = instead of LIKE for patterns
= looks for exact match only. You need LIKE for partial matching with wildcards.
WHERE name = 'R%'→WHERE name LIKE 'R%'
❌ Forgetting quotes around text
String/text values always need single quotes. Numbers do not. Mixing this up causes errors.
WHERE city = Delhi→WHERE city = 'Delhi'
❌ Confusing AND vs OR
AND is stricter (both conditions required). OR is broader (either condition works). Easy to mix up.
AND = fewer rows. OR = more rows.
❌ DELETE/UPDATE without WHERE
The most dangerous SQL mistake. Without WHERE, the operation applies to ALL rows in the table.
DELETE FROM Users→DELETE FROM Users WHERE id=5

🏆 The Master Query — All Concepts Together
SQL — Full Power Query
Copy
-- Top 5 unique cities of adult students from Delhi/Mumbai, sorted by marks
SELECT DISTINCT name, city, marks
FROM Students
WHERE age BETWEEN 18 AND 25
  AND city IN ('Delhi', 'Mumbai')
  AND name LIKE 'R%'
ORDER BY marks DESC
LIMIT 5;
