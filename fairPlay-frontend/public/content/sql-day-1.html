💬 What is SQL?

Structured Query Language

SQL (pronounced "sequel") is the language you use to talk to a database. Just like we use English to communicate with people, we use SQL to communicate with databases — to create tables, insert data, search for records, and update or delete information.

🗄️ Database & Table Structure

🗄️ What is a Database?

A database is an organized collection of data stored electronically. Think of it like a smart digital filing cabinet where data is stored in an orderly way so it can be easily found and used.

🎓 Example: A Student Database has all info about students — names, marks, attendance.

📋 What is a Table?

A table organizes data in rows and columns. Each row is one record (one student). Each column is one field (like name or age).

↔ Rows = Records
↕ Columns = Fields

🎯 Interview Questions & Answers
Q1
What is the difference between DELETE and DROP?
▶
DELETE removes specific rows from a table but keeps the table structure intact. It can be rolled back (undone).

DROP completely removes the entire table — both the data AND the structure. It cannot be rolled back.

💡 Think: DELETE = erase the content of a notebook. DROP = throw the entire notebook in the trash.
Q2
What is a Primary Key?
▶
A Primary Key is a column (or set of columns) that uniquely identifies each row in a table. It must be unique and cannot be NULL.

Example: In the Students table, id is the Primary Key — no two students can have the same id.

💡 It's like your Aadhar number — unique to only you.
Q3
Difference between WHERE and HAVING?
▶
WHERE filters rows before grouping. Used with SELECT, UPDATE, DELETE.

HAVING filters groups after GROUP BY. Used with aggregate functions like COUNT, SUM, AVG.

💡 Rule: WHERE works on raw rows. HAVING works on grouped/aggregated results.
Q4
What is the difference between TRUNCATE and DELETE?
▶
TRUNCATE removes ALL rows at once, is faster, and cannot be rolled back. It resets auto-increment counters.

DELETE removes rows one by one (can use WHERE), is slower for large tables, and can be rolled back.

💡 Use TRUNCATE to empty a table quickly. Use DELETE when you need to remove specific rows.
Q5
What are DDL, DML, and DQL?
▶
DDL (Data Definition Language) — defines the structure: CREATE, ALTER, DROP

DML (Data Manipulation Language) — changes the data: INSERT, UPDATE, DELETE

DQL (Data Query Language) — reads/fetches the data: SELECT

💡 DDL = blueprint, DML = construction/renovation, DQL = reading the blueprint.
